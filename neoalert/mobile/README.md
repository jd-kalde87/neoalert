# NeoAlert Mobile (Fase 2)

Aplicación móvil Flutter para operación en campo. **En espera** hasta completar la aplicación web.

## Módulos planificados (alineados con web)

- auth
- dashboard
- attendance
- maps
- incidents
- notifications

## Stack previsto

- Flutter 3.x
- Arquitectura por features (`lib/features/`)
- State management: Riverpod o Bloc
- Offline parcial + sincronización
- Push notifications

## Estructura objetivo

```
mobile/lib/
├── app/
├── core/
├── features/
└── shared/
```
